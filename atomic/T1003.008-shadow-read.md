# Atomic Test Plan — T1003.008 /etc/passwd and /etc/shadow

**Target rule**: `100100` — "Read of /etc/shadow"
**Data source**: auditd, key `credential_read`
**Endpoint**: `linux-ep-01`
**MITRE**: https://attack.mitre.org/techniques/T1003/008/

---

## Prereq (bắt buộc)

Audit rule phải có watch **read** trên `/etc/shadow`. File `auditd/wazuh-audit.rules` version mới (đã update ở Phase 4) chứa:

```
-w /etc/shadow -p rwa -k credential_read
```

Nạp lại trên VM:
```bash
# scp lại (nếu chưa)
scp auditd/wazuh-audit.rules gnid@192.168.154.166:/tmp/

# Trong VM
sudo cp /tmp/wazuh-audit.rules /etc/audit/rules.d/wazuh.rules
sudo augenrules --load
sudo auditctl -l | grep credential_read
# → -w /etc/shadow -p rwa -k credential_read
```

---

## Test — chạy tay (đơn giản)

**Trên linux-ep-01**:
```bash
sudo cat /etc/shadow > /dev/null
```

Hoặc chuyên nghiệp hơn (không log stdout ra terminal):
```bash
sudo cat /etc/shadow | head -c 0
```

**Expected**:
- audit event `type=SYSCALL ... key="credential_read" ... exe="/usr/bin/cat"`
- Rule `100100` fires level 12

**Verify**:
```
rule.id : "100100" AND agent.name : "linux-ep-01"
```

Hoặc query raw audit:
```bash
sudo ausearch -k credential_read | tail -20
```

**Cleanup**: no-op (test read-only).

---

## Biến thể

- **Attacker dùng `less` / `less /etc/shadow`** — vẫn trigger vì key `credential_read` watch mọi read
- **Attacker copy file: `sudo cp /etc/shadow /tmp/shadow.bak`** — trigger 2 key: `credential_read` (read source) + có thể trigger `T1074` staging nếu path `/tmp` match rule 100119
- **Attacker dùng `dd if=/etc/shadow of=/dev/null`** — trigger tương tự

Reference variant: Atomic official test `T1003.008` #1 dùng `cat /etc/shadow` — cùng logic.
